How the Automatic Newspaper Works: Sources, Architecture, and the Vision for Planetary Coverage

Choir Global Wire — Published May 30, 2026 — Updated continuously

The Choir Global Wire is a prototype of a new kind of publication: one that reads the world continuously, in many languages, and synthesizes what it finds into journalism every fifteen minutes. It is the first public artifact of Choir — a platform for building a live, global, open knowledge base that anyone can contribute to and query.

This page describes how it works, what sources it draws from, and why it is built the way it is.

The Core Loop

Every fifteen minutes, the system runs a three-phase cycle:

  1. Ingestion. A Go daemon polls 34 validated RSS feeds and one Telegram channel simultaneously, using HTTP conditional requests (ETag / If-Modified-Since) to avoid redundant downloads. New items are deduplicated by URL hash and stored in a local SQLite database.
  2. Synthesis. The 50 most recent items — drawn from sources across Asia, Africa, Latin America, the Middle East, and Europe — are passed to a large language model with a structured prompt. The model is instructed to write five original stories of 400–600 words each, with inline source citations, cross-regional analysis, and explicit attention to what non-English sources reveal that the English-language wire misses.
  3. Publication. The synthesized issue is stored in the database and immediately served to the frontend. The page auto-refreshes every fifteen minutes to display the latest issue.

The frontend is a static HTML page served by a lightweight Go HTTP server. It uses Pretext — a pure JavaScript library for text measurement without DOM reflow — for progressive layout enhancement, and marked.js for Markdown-to-HTML rendering in the browser.

Why Multilingual Sources Matter

The English-language wire — Reuters, AP, AFP — covers the world, but it covers it from a particular vantage point. Stories that matter deeply in Nairobi, São Paulo, Karachi, or Guangzhou often appear in English-language media days late, stripped of context, or not at all.

The Choir Global Wire is built on the premise that the most valuable signal is often the one that hasn't crossed the language barrier yet. When Welt am Sonntag breaks a story about US troop withdrawals from Europe, or when Folha de S.Paulo covers Brazil's response to US foreign policy in Portuguese, or when RFI Afrique reports on an Ebola outbreak in French before the English wire picks it up — that is the "extra signal" this system is designed to surface.

The synthesis model reads and reasons across all of these sources simultaneously, producing English-language journalism that cites its multilingual origins explicitly.

Technology Stack

  • Ingestion engine: Go 1.25, using gofeed for RSS/Atom parsing and standard net/http for Telegram web preview scraping.
  • Storage: SQLite via modernc.org/sqlite (pure Go, no CGo). Two tables: items (raw signals) and issues (synthesized output).
  • Synthesis: Python 3, OpenAI-compatible API with gpt-4.1-mini. Prompt engineering for structured journalism with inline citations and multilingual analysis.
  • Web server: Go net/http, serving static files and a JSON API (/api/latest).
  • Frontend: Vanilla HTML/CSS/JS. Pretext for text measurement without DOM reflow; marked.js for Markdown rendering; Google Fonts (Playfair Display, Source Serif 4, Inter).
  • Cycle runner: Python subprocess loop, 15-minute interval, orchestrating ingest → synthesis → publish.

What Comes Next

This is Version 0 — a proof of concept built in a single day. The roadmap includes:

  • More sources: Telegram public channels (regional conflict monitors, government feeds, investigative journalists), GDELT 15-minute event stream, Bluesky Jetstream, Wikipedia recent changes, arXiv preprints, prediction markets (Polymarket, Kalshi), maritime AIS data, and public health biosurveillance feeds.
  • Deeper multilingual coverage: Direct ingestion from Arabic, Chinese, Hindi, Swahili, and Hausa sources via RSSHub bridges for platforms like Weibo, Zhihu, and regional news aggregators.
  • Integration with Choir: Stories will become VTexts — versioned, citable, multimedia content objects — in the Choir knowledge base, retrievable by agents and linkable by users.
  • Cross-validation and adversarial evals: Multiple models checking each other's synthesis for factual consistency, source fidelity, and framing bias.
  • Custom domains: Private computers running their own instance of the Global Wire, filtered to their interests, published at their own domain.

The long-term vision is a planetary knowledge base: every noteworthy event on Earth, covered in the language it happened in, synthesized into a shared record that anyone can read, query, and contribute to.

Choir is open. The code lives at github.com/choir-hip/go-choir.
