# Cognitive Transform Portfolio

Last updated: 2026-05-14

Canonical skill file: [`../skills/cognitive-transform-portfolio/SKILL.md`](../skills/cognitive-transform-portfolio/SKILL.md)

A cognitive transform is an operator that changes the representation of a problem so new actions, risks, invariants, or values become visible.

Use the Cognitive Transform Portfolio when an agent is stuck, shallow, over-literal, audience-misaligned, or cargo-culting a framework. The skill is not a decorative theory menu. Select 2-5 transforms that can change the route, verifier, scope, evidence plan, or stopping condition.

## Core rule

```text
A transform that does not change action is commentary.
A transform that changes action is cognition.
```

## Primary transforms

The portfolio begins with two primary transforms because they are the most general:

1. **Audience-Level Translation** — accessibility through perspective shift.
2. **Depth Extraction / Esoteric Upgrade** — depth through recovery of the hidden load-bearing truth.

They are not mandatory for every use and should not be over-specified relative to the rest of the menu. They are primary because serious work fails either when the right idea cannot travel to the audience that must act, or when the idea becomes shallow enough to lose the underlying mechanism.

### Audience-Level Translation

Re-explain the same idea for a specific audience without losing the core structure.

Output shape:

```text
Audience:
Core idea:
Words to avoid:
Analogy:
Explanation:
What this audience can do with it:
What nuance is deferred:
```

### Depth Extraction / Esoteric Upgrade

Recover the deep mechanism beneath a familiar concept, slogan, framework, or commonplace.

Output shape:

```text
Concept:
Banal version:
Deep version:
Load-bearing variable:
Common failure mode:
Operational implication:
```
