# MissionGradient Choir-in-Choir Final Report

Date: 2026-05-12

Mission document: `docs/mission-gradient-choir-in-choir-2026-05-11.md`

## Result

The mission produced a deployed, evidence-backed bootstrap of the Choir self-development factory, but not a fully closed autonomous Choir-in-Choir loop.

Completed:

- Active desktop VM state now survives direct vmctl restarts and GitHub Actions staging deploys.
- Background worker VM creation, discard safety, mutable work, and patchset export are proven on staging.
- A platform shipper boundary exists. Worker contexts export manifests and patchsets without broad GitHub credentials; platform import can push a branch and trigger GitHub CI.
- VText product-path proofs cover prompt bar -> conductor -> VText -> researcher/super -> background worker VM -> patchset export -> VText current-state revision.
- VText mobile/tablet/desktop UX was improved enough to stop controls overlapping text and to preserve usable geometry.
- Trace, Settings, app registry, and theme configuration were cleaned up and verified through product-safe surfaces.
- A shared content substrate exists for URL import, extraction provenance, media classification, and app routing.
- First-pass content apps exist for PDF, EPUB, image, video/YouTube, audio, and podcast RSS.
- A publication path skeleton now defines the user desktop -> platform publishing desktop boundary.

Not completed:

- The product does not yet automatically invoke the platform shipper from a user trajectory and feed PR/CI/deploy provenance back into that same trajectory.
- Content apps were implemented as bootstrap code, not generated by Super in background VMs.
- The content apps are reader/player skeletons, not polished publishing-grade apps.
- Platform Dolt publication infrastructure is documented but not implemented.

## Invariant Audit

Preserved:

- Browser proof paths did not call `/api/agent/*`, `/api/prompts`, `/api/test/*`, `/internal`, or `/api/events`.
- Bare content references use product prompt-bar routing and server-owned `open_app` decisions, not browser-owned runtime metadata.
- Mutable code/file work is routed through Super/background worker VM proof paths where claimed.
- Worker VM export keeps GitHub push authority outside worker contexts.
- Trace is the evidence source for VText workflow causality, worker messages, tool calls, search stats, and worker VM export details.

Partially preserved:

- GitHub shipper provenance exists, but not yet as first-class Trace events inside the product trajectory.
- Content-app routing is production-shaped, but content-app implementation was local bootstrap work.
- URL import has provenance and bounded fetching, but still needs SSRF/private-network hardening before it is treated as secure.

Not yet exercised:

- Multiple parallel background VMs generating independent app changes and merging through the platform shipper.
- Promotion/rollback animation or active VM replacement flow.
- Platform publication from user embedded Dolt to platform Dolt.

## Residual Risks

- Product-invoked shipper is the main structural gap. Until that exists, the factory can export work and the platform can ship work, but the full deployed loop is not one continuous product trajectory.
- Search and extraction are functional but still early. The in-tree extractor is `readability_lite`, not Defuddle, and Obscura is not yet an acquisition rung.
- Content app UX is intentionally skeletal. EPUB, PDF, media metadata, fullscreen behavior, local upload handling, and transclusion hooks need deeper work.
- `/api/content/import-url` needs SSRF/private-network denylist hardening.
- Duplicate addressed deliveries were observed in the background-worker proof. Worker-update consumption needs idempotency/deduping.

## Next Optimization Target

The next MissionGradient run should target the continuous product loop:

`prompt bar -> VText -> researcher/Super -> background VM -> patchset export -> platform shipper -> GitHub PR/CI -> deployed staging proof -> Trace/VText provenance`

The critical value criterion is not "more apps." It is reducing divergence between the product trajectory and the shipping trajectory while preserving authority boundaries.

Concrete next work:

- Add product-invoked platform shipper handoff after background worker export.
- Emit shipper branch, PR, CI, and deploy provenance into Trace and back into the VText document.
- Add dedupe/idempotency for repeated worker-update deliveries.
- Harden URL import before expanding extraction.
- Port Defuddle and test Obscura as optional acquisition rungs.
- Continue content-app polish after the shipper path can carry app work safely.
